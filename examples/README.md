# Dyana Example Trajectories

These trajectories are intended for manual validation, documentation examples,
and slower integration smoke tests. They are larger than unit-test fixtures and
should not be used by the default fast test suite.

## Conventions

- Lengths are in Angstrom.
- Time spacing is the interval between stored trajectory frames.
- RDF example folders contain the scripted input log and the corresponding
  reference output generated by Dyana.
- The LAMMPS examples currently use orthorhombic boxes, matching Dyana's present
  trajectory-reader assumptions.

## RDF Examples

### Water O-H RDF

- Trajectory: `xyz/water128.xyz`
- Scripted input: `xyz/rdf/input.log`
- Reference output: `xyz/rdf/rdf_O_H.dat`
- Selection: compound 1 vs compound 1, reference atoms `O`, observed atoms `H`
- Bins: 1000
- Maximum distance: default 10.0 Angstrom

Run from the repository root:

```bash
python main.py examples/xyz/water128.xyz -i examples/xyz/rdf/input.log -l input.log
```

### Na-O RDF In NaClO4/Water

- Trajectory: `lammps/naclo4_h2o.lmp`
- Scripted input: `lammps/rdf/input.log`
- Reference output: `lammps/rdf/rdf_Na_O.dat`
- Selection: reference compound 3 atoms `Na`, observed compound 2 atoms `O`
- Bins: 1000
- Maximum distance: default 10.0 Angstrom

Run from the repository root:

```bash
python main.py examples/lammps/naclo4_h2o.lmp -i examples/lammps/rdf/input.log -l input.log
```

## Notes For Future Integration Tests

- Keep these examples out of the default fast unit-test suite.
- Add opt-in slow tests that compare key properties rather than full output
  files, unless the output format is intentionally being regression-tested.
- If the trajectory files are shortened or regenerated, update the frame counts
  and RDF reference outputs together.
